# 一、标识符作用域

## 1、作用域

```c
#include<stdio.h>
void func(){
	int n;
	n=100;
	printf("n in func is %d\n",n);
}
int main(){
	
	int n=0;
	printf("n in main is %d\n",n);
	func();
	printf("n in main is %d\n",n);
	return 0;
}
```

>在 main 函数中有一个变量名为 n 的变量，其值为0。而在函数 func 中，也有一个变量名为 n 的变量，并将其赋值为100。<br>
>运行结果如下：

<img width="399" height="162" alt="image" src="https://github.com/user-attachments/assets/877515ca-1433-4cba-a904-777de76d269e" />

>函数 main 中的变量 n 始终为0，而函数 func 中的变量 n 被赋值为100。<br>
>通过结果看来，函数 func 中的变量 n ，与函数 main 中的变量 n ，并不是同一个变量。下面，让我们引入作用域的概念，更加明确地描述并解释这个现象。


## 2、块作用域

>在源代码中，被花括号包括的代码形成一个代码块，若声明代码块内，适用范围从变量声明开始到代码块结束的那个花括号，这一段的使用范围成为块作用域。


>因此我们可以得出以下结论：多层级代码块形成的嵌套同名标识符作用域中，内层作用域将覆盖外层作用域。


>如果在代码块 B 后，再加一个 printf 。由于代码块 B 中的标识符 n 作用域已结束，覆盖效果将消除。那么，它使用的标识符 n 则指代代码块A开头声明的变量 n 。


```c
#include<stdio.h>
int main(){
	
	int n;
	printf("%d\n",&n);
	{
		int n;
		printf("%d\n",&n);
	}
	printf("%d\n",&n);
	return 0;
}
```

## 3、括号内参数的作用域

>函数可以形成带括号的块作用域。参数列表中声明的标识符，作用域为整个函数

```c
void printf(int n){  //作用域开始 
	//****
	return n;        //作用域结束 
}
```

>同理在for循环和while等地方，参数的作用域也是在花括号括起来的范围内

```c
for(int i=0;i<=100;i++){
	printf("%d",i);      //在作用域内 
}
printf("%d",i);          //在作用域外
```

## 4、文件作用域

>在代码块外声明的标识符，它的作用域从声明开始，直到该源文件结束。这种作用域被称作文件作用域

# 二、预处理指令

## 1、符号常量

>使用 #define 指令，将一个值使用一个符号替代。定义符号常量的公式：

```c
#define 符号常量 值（字符串类型）
```

>我们用下面这个程序来计算买一件商品，两件商品，三件商品分别需要多少钱？

```c
#include<stdio.h>
#define PRICE 3  //一件的价格为3 

int main(){
	
	int num;
	int total=0;
	//买一件
	num=1;
	total=num*PRICE;
	printf("num: %d  total: %d\n",num,total); 
	//买两件
	num=2;
	total=num*PRICE;
	printf("num: %d  total: %d\n",num,total); 
	//买三件
	num=3;
	total=num*PRICE;
	printf("num: %d  total: %d\n",num,total); 
	return 0;
}
```

>如果将单价涨价为4元，直接修改符号常量PRICE即可实现

```c
#include<stdio.h>
#define PRICE 4  //一件的价格为4

int main(){
	
	int num;
	int total=0;
	//买一件

	printf("num: %d  total: %d\n",num,total); 
	//买两件
	num=2;
	total=num*PRICE;
	printf("num: %d  total: %d\n",num,total); 
	//买三件
	num=3;
	total=num*PRICE;

}
```

## 2、预处理指令

>源代码中，以井号 # 开头的并不是C语言中的语句。它们属于预处理指令。<br>
>在代码被编译前，预处理器会先处理预处理指令，并根据预处理指令的意义修改C语言源码。<br>
>修改后的代码将另存为中间文件或直接输入到编译器。并不会保存到源文件中。所以，请放心，预处理器不会改动你的源文件。

## 3、#define预处理指令

>预处理指令 #define 的用法其实非常丰富，前面我们仅使用它定义了一些符号常量。现在，我们来了解它的完整的使用公式。

```c
#define 宏 替换体
```

>一旦预处理在程序中找到宏后，就会用替换体替换该宏。

>宏的命名规则遵循C语言标识符的命名规则：只能使用字母、数字、下划线，且首字符不能是数字。替换体不仅仅限于值，它的形式非常丰富，唯一的要求就是替换到代码后，你的代码还能正常通过编译

<img width="902" height="656" alt="image" src="https://github.com/user-attachments/assets/43a0a415-6db3-45da-a954-5c065ada5170" />


>从上面的例子可以看出，宏的替换是无差别的，它仅仅把代码当做文本来处理，遇到宏就替换为宏对应的替换体。



## 4、带参数的#define


>在 #define 中使用参数可以创建外形和作用与函数类似的宏函数。宏函数的使用公式如下：

```c
#define 宏(参数1，参数2，.... ，参数n) 替换体
```

>这个用法类似于函数，例如求a,b两个数的平均值

```c
#define MEAN(a,b) (a+b)/2
```

>在程序中，可以这样使用它

```c
int result;
result = MEAN(2,4);
```

>虽然由带参数的 #define 定义的宏函数，在使用方法上很像函数。但是，请再次注意，它的本质依然是将宏替换为对应的替换体。<br>

>由于宏函数仅仅是完成替换操作，将参数替换并拼接到替换体的表达式中。而不是先让参数运算得到结果后，再进行运算。因此，为了保证参数不被其他运算符优先级影响，请在参数两边加上括号。


# 三、typedef关键字

>这东西说白了就是给某个类型取一个别名，如果我们要求程序需要满足在不同的平台上均能正确的运行，不会因为整型数据范围不同而产生数据溢出。那么，我们可以为整型取一些别名。


<img width="938" height="649" alt="image" src="https://github.com/user-attachments/assets/31950230-cd4c-4b6e-bb47-06abfd8b6294" />


>我们将整型类型使用别名替代，在不同平台下编译时，仅需要更改别名对应的实际类型，即可避免不同平台下，整型数据范围不同而产生数据溢出。<br>

>除了使用 #define ，C语言还提供了 typedef 关键词用于定义类型别名。语法和define几乎一样，例如int32_t的定义如下：

```c
typedef int int32_t ;
```

>这样就将原来的变量名变成了int32_t这种了

```c
#include<stdio.h>
typedef int int32_t; 
int main(){
	
	int32_t n=1291823;
	printf("%d",n);
	return 0;
}
```

<img width="406" height="162" alt="image" src="https://github.com/user-attachments/assets/83c8f2e2-b69f-448c-94a7-64854bb55893" />

>别名如果定义在代码块中，那么它就具有块作用域。别名的作用域从别名声明开始，直到包含声明的代码块结束。函数 add 不在别名的作用域内。因此，函数 add 中无法使用别名 int32_t 。


<img width="909" height="599" alt="image" src="https://github.com/user-attachments/assets/5487e79c-d441-4677-82e0-879e908595bb" />


>如果定义在块外，那么它具有文件作用域。别名的作用域从声明开始，直到该源文件结束。作用域内均可使用别名 int32_t 。

<img width="902" height="633" alt="image" src="https://github.com/user-attachments/assets/c8ed2c7d-9888-4b39-9037-4a14a2424771" />

>typedef 也被经常用于结构体，加个这个上去，在申明结构体变量的时候就不需要加struct了。例如，有一个人员信息结构。

```c
#include<stdio.h>

typedef struct{
	char name[20];
	int gender;
	double weigh;
	double heigh;
}person; 

int main(){
	
	person p; //这样就无需加上struct了 
	return 0;
}
```

>使用 typedef 时要注意， typedef 并没有创建任何新类型，它只是为某个已存在的类型增加了一个方便使用的别名。


# 四、typedef和define的区别

>- typedef 只能用于给类型取别名，不能用于值。
>- typedef 由编译器解释，而不是预处理器。
>- typedef 在某些情况下，比 #define 更合适。
