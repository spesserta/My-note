>整形变量表达数值范围的原理：<br>
>char，short，int，long，long long分别占用了1，2，4，4，8个字节。而每个字节由8个晶体管组成， 每个晶体管状态我们称之为位。那么char，short，int，long，long long分别占用了8，16，32，32，64 位。<br>
>太多的位不利于我们理解原理，我们暂时把问题简化一下，试试看位数减少到3。然后，分析3位的组合，它能表示多大范围的数值呢？<br>

<img width="881" height="460" alt="image" src="https://github.com/user-attachments/assets/dd96fc49-f667-4f45-929f-057db3dcb641" />

> 三位二进制组成的数据类型，可以表达2的3次方也就是8个数值。如果从0开始，那么可以表达从0到7的 数据范围。那么我们可以得出以下结论：<br>
>如果不考虑负数，那么整型数据类型可以表达的数据范围是：<br>
>假设，位数为n，则数据范围从【0】开始，到【2的n次方-1】的数值范围。<br>
>例如3位2进制，从0开始到2的3次方-1=7<br>
>负数怎么办？那我们需要拿出一个位来作为符号位。用来表示这个数据是正数 还是负数。在IEEE标准中，这个符号位存在于二进制的最高位。让我们用三位二进制来示范这种情况.<br>


<img width="862" height="456" alt="image" src="https://github.com/user-attachments/assets/e1b8d8f1-8e55-4ea6-8281-708a279fa6c0" />


加上符号之后，现在取值范围变为负4到3了。红框中的为最高位，最高位为1的表示负数。你可能会觉 得有点奇怪，为什么3的二进制是011，而负3却是101呢？如果简单的加一个符号位，为什么不用111 呢？那我们看看如图中所示的3与负3相加的运算结果：<br>

<img width="798" height="398" alt="image" src="https://github.com/user-attachments/assets/1ef34fbd-4958-4a78-865c-58b35335e7f4" />


>用101来表示负3与用011表示的正3相加。结果为1000，但是由于仅有3位二进制来保 存数据，最高位1被丢弃了。结果为000，居然得到了正确的结果0。为什么会这样呢？<br>
>数值的补码表示：<br>
>假设有一个时钟是一个圆被分成了12个点，让我们假设这个时钟一步只能走一个整点。那么这个时钟只有12种不同的模式，我们把12称之为时钟的模。 现在指针指向了5点，我们要让指针回到0点。一个办法是直接回退5个小时（5-5）、另一个方法就是前进7小时（5+7）。在第二种办法中，5+7=12，而12刚好为时钟的模，时钟指向12的同时，也正好指向了0。 要让指针回到0点，只需要让它加上模与当前的时间的差即可。<br>
>因此，指针回退5小时与指针前进7小时是等价的。我们可以用指针前进来代替指针后退。 让我们将这种思想带入到上面讨论的三位二进制当中。三位二进制能表示8中不同的模式，因此它的模 为8。要让3回到0，我们可以让3减去3，也可以让3加上模与3的差，即8-3=5。因此，我们可以把-3在三 位二进制中用5的二进制101来表示。<br>
>这种将用加法来等效减法的二进制表示法被称之为补码表示法。<br>
>正数的补码就是其二进制本身。 而正数对应的负数的补码为：(模 - 正数)的二进制。<br>

<img width="908" height="497" alt="image" src="https://github.com/user-attachments/assets/57981cb5-b2c0-496e-acb5-a82a09a46f5f" />


类似于钟表仅需要向前走就可以实现减法，计算机的电路设计中，也只需要设计加法电路。极大地 简化了计算机内部电路的复杂程度。

# 十一、字符与字符串

## 1、ASCII码

>在C语言中字符常量是用单引号括起来的，例如'A','a','t'。
>字符串是由双引号来括起来的，例如printf("Hello world");
>注意printf('a');这样是错误的，C语言规定printf只能用双引号输出字符串，单引号需要使用字符的占位符%c来输出

```c
#include<stdio.h>
int main() {
	printf("%c%c%c%c%c %c%c%c%c%c",'H','e','l','l','o','W','o','r','l','d');
}
```

>注意字符常量和字符变量只占用1个字节的空间也就是8个晶体管，表示的数值范围是0-255，为了将数值转换成相应代表的字符，ASCII码就出现了，以下就是ASCII码表

<img width="747" height="925" alt="image" src="https://github.com/user-attachments/assets/ddcd11ce-7e4c-4d81-88ad-35ceb5957510" />

>常用的ASCII码需要牢记，例如a的ASCII码是97，A的ASCII码是65，当然我们也可以通过改变printf输出的格式得出字符的ASCII码

```c
#include<stdio.h>
int main() {
	char c1 = 'a';
	char c2 = 'A';
	char c3 = 'B';
	printf("a的ASCII码是%d\n", c1);
	printf("A的ASCII码是%d\n", c2);
	printf("B的ASCII码是%d\n", c3);
	return 0;
}

```

## 3、转义字符

> \+ASCII码 被称作转义字符。 转义字符虽然写法上有多个字符，实际上它对应ASCII表中的一个字符。 由于直接在字符串中写0会被认为是字符0，而 \数值 这种写法，可以让我们输入数值0。但是，这个数 值不是用十进制来表示的，而是用八进制。 那我们尝试直接使用数值来打印字符串Hello，注意斜杠后面跟着的是八进制数值。

```c
#include<stdio.h>
int main() {
	//'h'的ASCII码的8进制的110，'e'的145，以此类推
	printf("\110\145\154\157");
}
```
