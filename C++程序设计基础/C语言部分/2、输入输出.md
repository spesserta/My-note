## 5、占位符的类型和数量需要和后边的参数类型和数量相对应

<img width="907" height="520" alt="image" src="https://github.com/user-attachments/assets/2056e57f-5359-47b2-96c8-852b8e22d151" />


>整形类型的占位符：<br>
>Printf 是一个可变参数函数，在C语言中将参数传入函数的可变参数中，变量会发生自动类型提升。<br>
>对于有符号位的整型 char ， short ，传入 printf 的可变参数时，会被提升为 int 。而比 int 更高级的 整型则不发生变化。<br>

<img width="898" height="420" alt="image" src="https://github.com/user-attachments/assets/3c03372e-e9b8-4952-960f-3cee9618063f" />


>所以，在处理 char ， short ， int 时，均可使用 %d 来占位。 而在Visual Studio中 int 与 long 的范围一致，按理来说也可以使用 %d 来占位。 但是为了程序的可移植性，在切换到别的平台下时， int 和 long 有可能不一致。 所以最好使用 %ld 来为 long 占位。更高级的 long long 则需要使用 %lld 来占位。<br>
>结论： char，short，int使用 %d 。 long使用 %ld 。 long long使用 %lld 。<br><br>

>同理，对于无符号位的整型 unsigned char ， unsigned short ，传入 printf 的可变参数时，会被提升 为 unsigned int 。比 unsigned int 更高级的整型则不发生变化。

<img width="912" height="430" alt="image" src="https://github.com/user-attachments/assets/4b61281b-2b74-4b3a-a7b1-08c0edae34d5" />

>对了，无符号整型，需要将 d 替换成 u 表明最高位不被看作符号位，而是数据位。<br>
>结论： unsigned char，unsigned short，unsigned int使用 %u 。 unsigned long使用 %lu 。 unsigned long long使用 %llu 。<br>

>浮点数的话float会被提升为double，double本身不变。<br>
>结论： float，double均使用 %f 。<br>


# 二、printf函数的转换规则

>转换规范以%百分号开始，依次具有下面这些元素： 
>1. 零个或多个标志字符(-,+,0,#)。
>2. 一个可选的十进制整数常量表示的最小字段宽度。
>3. 一个可选的用点号表示的精度范围，它的后面可以跟一个十进制整数。
>4. 一个可选的长度指示符，可以用下列字母组合之一来表示：h、hh、l、LL，z。
>5. 由单个字符表示的转换操作，取自下面这个集合:c、d、e、E、f、i、o、s、u、x、X。

## 3、精度

>转换操作可以制定一个可选的精度范围，用一个点号以及它后面一个可选的十进制正数表示。 精度范围用于控制：<br>
>1. d、i、o、u、x、X所打印的最小数字位数，不足将补齐。 <br>
>2. e，E，和f转换中小数点右边的数字位数。

```c
#include <cstdio>

int main()
{
    int un = 123;
    double df = 123.456789;
    printf("%.6d\n",un);   //限制数值为6位，多删少补
    printf("%f\n", df);   //不限制的浮点数
    printf("%.0f\n", df);  //小数点后的数字个数限制为0
    printf("%.4f\n", df);  //小数点后的数字个数限制为4
    return 0;
}
```

## 1、将输入字符串和第一个参数进行匹配

<img width="910" height="214" alt="image" src="https://github.com/user-attachments/assets/68ef8ead-a292-4840-b924-46e166a4a6f8" />


>子串"1" 对应转换规范 "%hhd"  <br>

>子串 "4" 对应转换规范 "%ld"<br>

# 五、输入输出缓存

>生活中我们往往会将衣服集满了一桶再洗，而不是只有一两件脏衣服也启动洗衣机 导致浪费水电资源。 计算机中也有类似的思想，减少不必要的操作开销。
>在向控制台打印字符时，程序会先将需要打印的字符串放在输出缓存区中，到特定时刻，再一起显示到 控制台。 缓存区就类似于存放脏衣服的桶子，里面存放了一定量的数据，等待批量处理。的数据发送至目的地并清空缓存，这一行为称之为 刷新缓存

<img width="914" height="376" alt="image" src="https://github.com/user-attachments/assets/9cee8828-0554-43ec-a4e0-bda753fdacdf" />

>在windows系统下，使用printf后，数据被写入到输出缓存区。随后，立即刷新缓存区。<br>
>但是在linux系统下，使用printf后，数据被写入到输出缓存区。后续的printf会在缓存区中累积数据。直到程 序结束才刷新缓存区。


## 1、缓存可以分为行缓存和完全缓存

>输入输出缓存属于行缓存，即一行结束后必须刷新缓存。 另外，还有一种缓存形式为完全缓存，这种缓存的形式需要等到整个缓存区被填满，才会刷新缓存。对于行缓存，如果我们想要刷新缓存，只要将一行结束（'\n'）即可。
>因此在linux系统中如果想逐行显示的话，需要在每一行后边加上\n

## 2、输入缓存区

>类似于输出函数（如printf，putchar）存在输出缓存，输入函数（如scanf，getchar）也存在输入缓存。 并且这些输入函数属于阻塞函数，当输入缓存区没有内容时。程序将阻塞在输入函数中，等待用户从键 盘键入字符，并按回车确认。 当我们按下了回车键（即换行，存储为'\n'），输入的字符串将进入输入缓存区。 接下来，输入函数将从输入缓存区获取字符，删除缓存区中已获取的字符，并解除阻塞状态继续执行代 码。

<img width="898" height="370" alt="image" src="https://github.com/user-attachments/assets/950b59f5-17de-4884-95f5-97cc9d55b20b" />

>以gethcar和putchar为例

```c
#include <cstdio>

int main()
{
    char c1, c2;
    c1 = getchar();
    putchar(c1);
    c2 = getchar();
    putchar(c2);
    return 0;
}
```

<img width="400" height="185" alt="image" src="https://github.com/user-attachments/assets/13903c84-d37d-49e3-9b80-2f3ed09db8cf" />


>程序运行到第一个getchar，由于输入缓存区没有数据，getchar进入阻塞状态，等待用户输入。 在输入'A'并按回车后，"A\n"进入了输入缓存区。 第一个getchar获取了字符'A'，解除阻塞状态，继续执行。接着用putchar打印c1。 执行到第二个getchar时，由于缓存区内仍有字符'\n'。getchar无需阻塞等待输入，直接获取了字符'\n'， 接着用putchar打印c2。 换行符'\n'被当做了第二个getchar的输入，显然，我们不希望这样。我们可以用一个getchar来吸收第一 次输入的'\n'。

```c
#include <cstdio>

int main()
{
    char c1, c2;
    c1 = getchar();
    putchar(c1);
    getchar(); //用于吸收回车键的'\n'
    c2 = getchar();
    putchar(c2);
    return 0;
}
```

<img width="433" height="203" alt="image" src="https://github.com/user-attachments/assets/8754a22c-695e-4e39-a4a4-fd5bec0fb9b0" />

>因此输入输出字符串除了常用的printf和scanf，还有更加灵活的getchar和putchar

```c
#include <cstdio>

int main()
{
    char str[100];
    int i = 0;
    while (i < 100 - 1) {  //最多输入100个字符
        char c;
        c = getchar();
        str[i++] = c;
        if (c == '\n') {  //遇到换行符也停止输入
            break;
        }
    }
    str[i] = '\0';
    printf(str); //还记得不，printf括号里本质是一个字符串，因此字符串不用加“”
    return 0;
}
```
