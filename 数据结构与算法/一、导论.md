# 一、数据结构的基本概念

>- 数据：能被计算机识别，处理的符号的集合，可以看成一个整体<br>
>- 数据元素：用于描述一个个体，由几个数据项组成<br>
>- 数据项：描述数据的最小单位，不可以再分割<br>
>- 数据对象：具有相同性质的数据元素的集合，是数据的子集<br>
>- 数据结构：描述数据元素间的关系

# 二、数据结构三要素

## 1、逻辑结构：数据元素之间的逻辑关系 

  >- 集合结构：无关系 <br>
	>- 线性结构：一对一<br>
	>- 树形结构：一对多<br>
  >- 图形结构：多对多<br>
	
## 2、存储结构（物理结构）：在计算机内部表示出数据元素之间的逻辑关系  

>- 顺序存储：物理地址和逻辑地址相同<br>
>- 链式存储：物理地址和逻辑地址不一定相同<br>
>- 索引存储：类似目录，每个数据有对应的地址表示<br>
>- 散列存储：哈希表
	
## 3、数据的运算：针对某种逻辑结构，结合实际需求，定义基本运算  

>- 增加<br>
>- 删除<br>
>- 修改<br>
>- 查找 <br>
>- 排序<br>
	
# 三、数据类型和抽象数据类型

>- 数据类型分为<br>
>- 原子类型（他的值不可以再分，比如int float）<br>
>- 结构类型（值可以再分，比如数组结构体）<br>
>- 抽象数据类型  是对数据结构逻辑结构的描述，以及数据的运算在计算机内部的实现<br>
	
# 四、程序=数据结构+算法

>- 数据结构是如何用数据正确的描述现实世界并存入计算机<br>
>- 算法是对特定问题求解步骤的一种描述
	
# 五、算法的特性：

>- 有穷性：有限个步骤<br>
>- 确定性：每条指令必须有确切的含义，相同输入只能得出相同输出，没有歧义<br>
>- 可行性：可以通过已经实现的基本运算执行有限次来实现<br>
>- 输入和输出：算法输入可以0个或多个，输出不可以0个，只能1个或多个
	
# 六、好算法的特性：

>- 正确性：能解决问题<br>
>- 可读性：好理解，可以加注释增加可读性<br>
>- 健壮性：可以适应用户输入的极端情况<br>
>- 高效率：时间复杂度低<br>
>- 低存储量：空间复杂度低


# 七、时间复杂度的计算（事前统计运行时间）

>- 时间复杂度是指算法时间开销T与问题规模n的关系T=T(n)得出表达式后可以简化，找最大的项<br>
>- 复杂度排序O(1)<O(log2n)<O(n)<O(nlog2n)<O(n平方)<O(n3)<O(2n次方)<O(n的阶乘)<O(n的n次方)<br>

>-最坏时间复杂度:考虑输入数据“最坏”的情况<br>
>-平均时间复杂度:考虑所有输入数据都等概率出现的情况<br>
>-最好时间复杂度:考虑输入数据“最好”的情况
