### 一、为什么分层
>分层可以将把数据 → 业务 → 界面 分开，不乱、好维护、好扩展。 <br>
>就像家里： <br>
>厨房（存食材）= 数据库层 <br>
>客厅（处理食材、做饭）= 业务逻辑层 <br>
>餐厅（端给你吃）= 界面层 <br>
>如果混在一起，厨房、客厅、餐厅都在一个房间，那肯定乱死。


### 二、经典三层架构

#### 1、数据访问层 DAL（Data Access Layer）

##### 该层和数据库打交道

>- 打开连接
>- 执行 SQL
>- DataSet、DataAdapter、增删改查
>- 别的啥都不干

#### 2、业务逻辑层 BLL（Business Logic Layer）

##### 该层用来处理规则

>- 验证数据是否合法
>- 年龄不能为负
>- 用户名不能为空
>- 不能重复插入
>- 调用 DAL 层

#### 3、表示层 UI（User Interface）

##### 这层用来显示、接收输入

>- 控制台、WinForm、WPF、网页
>- 点击按钮
>- 显示数据
>- 调用 BLL 层


### 三、三层之间的关系（自上而下调用）

```
UI 界面层
   ↓（调用）
BLL 业务逻辑层
   ↓（调用）
DAL 数据访问层
   ↓（操作）
数据库
```
>注意这三层绝对不能跨层，例如UI不能直接访问数据库

```
你的项目
├── UI层（Program.cs 控制台界面）
├── BLL层（TestManager.cs 处理业务）
├── DAL层（TestService.cs 操作数据库）
└── Models层（Test.cs 实体类）
```

```
UI 只管显示
BLL 只管规则
DAL 只管数据库
Model 代表表结构
```
